# Infrastructure and Repositories

## Infrastructure

DataGen’s infrastructure model includes:

- workstations and servers
- software inventory
- network assets
- telephony assets
- application service hosting relationships

That makes it possible to support labs where hosts, installs, and operational ownership matter.

## Repositories and collaboration

Current repository and collaboration output can include:

- databases
- file shares
- SharePoint or collaboration sites
- folders, libraries, pages, and channels
- direct and inherited permission shapes
- deny entries and unique-permission exceptions

## Why this matters

These surfaces make the generated world useful for:

- repository discovery validation
- collaboration permission demos
- server-to-app or repo-to-app mapping scenarios
- access and ownership exploration in labs
