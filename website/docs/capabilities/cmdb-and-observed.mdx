# CMDB and Observed Views

CMDB is now a native capability rather than an export-side afterthought.

## Canonical configuration items

The current generator can project canonical configuration items from:

- business services
- applications and platforms
- servers and devices
- repositories and collaboration systems
- identity stores and major cloud services

## Source-facing record types

Generated CMDB and discovery-style views can represent multiple synthetic source systems, including:

- CMDB
- auto-discovery
- service catalog
- spreadsheet import

## Realistic drift

The product now supports the kinds of deviations teams regularly see in real environments:

- missing owners
- stale owners
- partial classification
- discovery-only records
- service-catalog entries without formal deployment
- canonical items missing from CMDB-facing views

This is especially valuable when you are validating reconciliation logic or demonstrating why source quality matters.
